## FitnessWon Backend – PostgreSQL Schema

Create these tables in your PostgreSQL database (or Supabase).  
Run the SQL statements in order.

---

### 1. `user_profiles`

Stores all the core data used to personalize AI responses.

```sql
CREATE TABLE IF NOT EXISTS user_profiles (
    user_id           uuid PRIMARY KEY, -- or text if you prefer
    name              text NOT NULL,
    age               integer NOT NULL,
    gender            text NOT NULL, -- e.g. 'male', 'female', 'other'
    height_cm         numeric(5,2) NOT NULL,
    weight_kg         numeric(5,2) NOT NULL,
    goal              text NOT NULL, -- e.g. 'fat_loss', 'muscle_gain'
    fitness_level     text NOT NULL, -- e.g. 'beginner', 'intermediate', 'advanced'
    dietary_preference text NOT NULL, -- e.g. 'omnivore', 'vegetarian', 'vegan'
    target_weight_kg  numeric(5,2) NOT NULL,
    timeline_weeks    integer NOT NULL,
    activity_factor   numeric(4,2) NOT NULL DEFAULT 1.40,
    injuries          text,
    created_at        timestamptz NOT NULL DEFAULT now(),
    updated_at        timestamptz NOT NULL DEFAULT now()
);
```

Recommended trigger in Postgres/Supabase to auto-update `updated_at` (optional).

---

### 2. `chat_history`

Stores AI ↔ user conversations for context and review.

```sql
CREATE TABLE IF NOT EXISTS chat_history (
    id           bigserial PRIMARY KEY,
    user_id      uuid NOT NULL,
    role         text NOT NULL CHECK (role IN ('user', 'assistant')),
    content      text NOT NULL,
    context_type text NOT NULL DEFAULT 'general',
    created_at   timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_chat_history_user_created_at
    ON chat_history (user_id, created_at DESC);
```

This table is used in:
- `/ai/chat` (load last 10 messages, save new messages).

---

### 3. `workout_plans`

Stores 7-day workout plans generated by Gemini as JSON.

```sql
CREATE TABLE IF NOT EXISTS workout_plans (
    id         bigserial PRIMARY KEY,
    user_id    uuid NOT NULL,
    plan_json  jsonb NOT NULL, -- full workout plan from Gemini
    created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_workout_plans_user_created_at
    ON workout_plans (user_id, created_at DESC);
```

Used in:
- `/ai/generate-workout-plan` (insert new plan).
- `/plans/workout/{user_id}` (fetch latest plan).

---

### 4. `diet_plans`

Stores 7-day meal plans with macros and TDEE.

```sql
CREATE TABLE IF NOT EXISTS diet_plans (
    id         bigserial PRIMARY KEY,
    user_id    uuid NOT NULL,
    tdee       numeric(6,1) NOT NULL,
    plan_json  jsonb NOT NULL, -- full diet plan from Gemini
    created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_diet_plans_user_created_at
    ON diet_plans (user_id, created_at DESC);
```

Used in:
- `/ai/generate-diet-plan` (insert new plan).
- `/plans/diet/{user_id}` (fetch latest plan).

---

### 5. `body_measurements`

Stores circumferences / skinfolds / other body metrics over time.

```sql
CREATE TABLE IF NOT EXISTS body_measurements (
    id              bigserial PRIMARY KEY,
    user_id         uuid NOT NULL,
    date            date NOT NULL DEFAULT CURRENT_DATE,
    neck_cm         numeric(5,2),
    shoulders_cm    numeric(5,2),
    chest_cm        numeric(5,2),
    waist_cm        numeric(5,2),
    hips_cm         numeric(5,2),
    thigh_cm        numeric(5,2),
    arm_cm          numeric(5,2),
    body_fat_pct    numeric(5,2),
    notes           text,
    created_at      timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_body_measurements_user_date
    ON body_measurements (user_id, date DESC);
```

Used in:
- `/ai/body-analysis` to analyze progress trends.

---

### 6. `check_ins`

Daily/weekly subjective check-ins (energy, sleep, stress, soreness, adherence).

```sql
CREATE TABLE IF NOT EXISTS check_ins (
    id            bigserial PRIMARY KEY,
    user_id       uuid NOT NULL,
    date          date NOT NULL DEFAULT CURRENT_DATE,
    mood          text NOT NULL,
    energy        text NOT NULL,
    sleep_hours   numeric(4,2) NOT NULL,
    stress_level  text NOT NULL,
    soreness      text NOT NULL,
    adherence     text NOT NULL, -- e.g. 'perfect', 'ok', 'poor'
    notes         text,
    created_at    timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_check_ins_user_date
    ON check_ins (user_id, date DESC);
```

Used in:
- `/progress/check-in` (insert entries).
- `/ai/body-analysis` (load history).

### 7. `weight_history`

Tracks body weight over time (for trend analysis and TDEE tuning).

```sql
CREATE TABLE IF NOT EXISTS weight_history (
    id         bigserial PRIMARY KEY,
    user_id    uuid NOT NULL,
    date       date NOT NULL DEFAULT CURRENT_DATE,
    weight_kg  numeric(5,2) NOT NULL,
    created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_weight_history_user_date
    ON weight_history (user_id, date DESC);
```

Used in:
- `/progress/weight` (insert entries).
- `/ai/body-analysis` (load history).

---

### 8. `progress_photos`

Stores references to progress photos (URL only; actual bytes live in storage, e.g. Supabase Storage).

```sql
CREATE TABLE IF NOT EXISTS progress_photos (
    id         bigserial PRIMARY KEY,
    user_id    uuid NOT NULL,
    photo_url  text NOT NULL,
    taken_at   timestamptz NOT NULL DEFAULT now(),
    created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_progress_photos_user_taken_at
    ON progress_photos (user_id, taken_at DESC);
```

Used in:
- `/progress/photo` to save photo URL references.

---

### 9. `workout_plan_completions`

Tracks which workout days from a given plan have been completed.

```sql
CREATE TABLE IF NOT EXISTS workout_plan_completions (
    id          bigserial PRIMARY KEY,
    user_id     uuid NOT NULL,
    plan_id     bigint NOT NULL REFERENCES workout_plans(id) ON DELETE CASCADE,
    day         text NOT NULL, -- e.g. 'Day 1', 'Monday', etc.
    completed_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_workout_plan_completions_user_plan
    ON workout_plan_completions (user_id, plan_id);
```

Used in:
- `/plans/workout/{user_id}/complete-day` to mark a day finished.

---

### 10. `supplement_logs`

User supplement intake history.

```sql
CREATE TABLE IF NOT EXISTS supplement_logs (
    id         bigserial PRIMARY KEY,
    user_id    uuid NOT NULL,
    name       text NOT NULL,
    dose       numeric(8,2) NOT NULL,
    unit       text NOT NULL, -- e.g. 'mg', 'g', 'capsules', 'scoops'
    timing     text NOT NULL, -- e.g. 'pre-workout', 'with breakfast'
    notes      text,
    created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_supplement_logs_user_created_at
    ON supplement_logs (user_id, created_at DESC);
```

Used in:
- `/supplements/log` (insert entries).
- `/supplements/log/{user_id}` (history).

---

### 11. (Optional) auth tables

Auth is very minimal in this version (only basic health endpoints).  
If you integrate Supabase Auth, you’ll typically rely on Supabase’s own `auth.users` table and won’t need to create it manually.
